# Capacitaciones Demo Items

Lista simple de funcionalidades y decisiones de negocio para contar en la demo del modulo de capacitaciones.

1. Gestion completa de capacitaciones desde panel admin.
2. Separacion entre listado de capacitaciones y edicion de una capacitacion puntual.
3. Creacion de nuevas capacitaciones.
4. Edicion de capacitaciones existentes.
5. Eliminacion de capacitaciones.
6. Previsualizacion de una capacitacion sin salir del panel admin.
7. Estado de publicacion de cada capacitacion.
8. Solo las capacitaciones publicadas aparecen del lado usuario.
9. Cada capacitacion debe tener al menos un modulo.
10. Cada modulo puede tener contenido propio.
11. Cada modulo acepta links de YouTube.
12. Cada modulo acepta archivos adjuntos.
13. Cada modulo tiene una prueba obligatoria.
14. Cada capacitacion tiene una evaluacion final obligatoria.
15. Las pruebas de modulo se editan por separado para no mezclar contenido con evaluacion.
16. La evaluacion final tambien se edita por separado.
17. Soporte de preguntas `multiple choice` con una unica respuesta correcta.
18. Soporte de preguntas `verdadero/falso`.
19. Configuracion de porcentaje de aprobacion por prueba.
20. Configuracion de duracion maxima por prueba.
21. Configuracion de cantidad de preguntas a mostrar en pruebas de modulo.
22. Colapso y expansion de modulos para reducir scroll.
23. Colapso y expansion de preguntas dentro de las pruebas.
24. Indicadores visuales por seccion: datos generales, modulos y evaluacion final.
25. Indicador de estado por modulo: completo o pendiente.
26. Motivo visible de por que un modulo esta pendiente.
27. Resumen visual de recursos cargados por modulo.
28. Resumen visual de datos de la prueba del modulo.
29. Proteccion contra salir con cambios sin guardar.
30. Aviso al volver, navegar, refrescar o cerrar pestaña si hay cambios pendientes.
31. Acciones de guardado mas claras desde el editor.
32. Posibilidad de ver la capacitacion desde dentro de la misma edicion.
33. Capacitaciones de ejemplo listas para cargar en base para demo.
34. Validacion para que una prueba vacia no cuente como completa.
35. Organizacion del editor en tres bloques claros: datos generales, modulos y evaluacion final.
